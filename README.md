# Tableau Dashboard: Мониторинг эффективности кол-центров службы поддержки

 **Интерактивный дашборд для руководителей кол-центров провайдера связи, разработанный на основе методологии Dashboard Canvas и серии бизнес-интервью.**

---

## Бизнес-контекст и задача

В компании по проведению интернета функции контроля клиентского сервиса разделены между **четырьмя руководителями кол-центров**. У каждого из них были свои изолированные требования к отчетности. 

**Цель проекта:** Собрать единый инструмент мониторинга, который закроет потребности всех четырех стейкхолдеров, позволит оперативно отслеживать качество работы операторов, выполнение нормативов (SLA) и уровень удовлетворенности клиентов (CSAT).

Для фиксации требований и метрик на этапе проектирования был разработан **Dashboard Canvas** *(файл доступен в репозитории)*.

## Функционал дашборда и ключевые блоки

Дашборд разделен на логические зоны для верхнеуровневого контроля и глубокого бурения (Drill-down) в проблемы:

1. **Главные KPI (Бизнес-мониторинг):** Агрегированные показатели **SLA** и **CSAT** с динамикой по дням. Позволяют мгновенно оценить общую ситуацию.
2. **Ресурсное планирование:** График нагрузки в течение дня, сопоставляющий фактическое количество операторов на смене с плановым. Помогает выявлять недогруз/перегруз смен.
3. **Географический разрез:** Тепловая карта звонков по штатам для локализации региональных всплесков обращений (например, при массовых сбоях связи).
4. **Операционный контроль (Операторы):** Рейтинг сотрудников по уровню CSAT для выявления лидеров и отстающих.
5. **Actionable-блок (Работа над ошибками):** Таблица «Звонки с самым низким CSAT» с прямыми ссылками на записи разговоров (`Call_id`) и тональностью общения для оперативного разбора конфликтных ситуаций супервайзером.

## Использованные инструменты и навыки в Tableau

* **Проектирование:** Разработка UI/UX макета по методологии Dashboard Canvas.
* **Фильтрация и интерактивность:** Настройка сквозных фильтров по датам, операторам, темам обращений и кол-центрам (Chicago/IL и др.) для кастомизации отчета под конкретного руководителя.
* **Работа с гео-данными:** Построение Heatmap (карт плотности) на базе встроенных карт Mapbox.
* **Продвинутая визуализация:** Использование комбинированных графиков (Dual Axis), референсных линий (Reference Lines) для отображения плановых и фактических значений.
